# 스페이스 배틀 플릿 (Space Battle Fleet)
## 게임 컨셉 기획서

## 1. 핵심 유저 경험 정의

### 한 문장 정의

> **"복잡한 조작 없이 수백 기의 함대를 지휘하며, '내 영토가 넓어지는 정복의 쾌감'과 '한 번의 판단 착오로 전세가 뒤집히는 줄다리기 긴장감'을 동시에 느끼는 경험."**

### 유저가 느끼는 경험의 흐름 (Emotional Journey)

이 게임은 플레이어에게 **세 단계의 감정 곡선**을 선사합니다.

| 단계 | 게임 시점 | 제공하는 감정 | 구체적 상황 |
|------|-----------|--------------|-------------|
| **① 고독한 개척** | 초반 (0~3분) | 미지에 대한 긴장 + 탐색 욕구 | 광활한 우주의 왼쪽 끝, 단 2개의 행성만 가진 채 시작. 카메라를 움직이면 회색 미지의 행성들과 저 멀리 붉게 빛나는 적의 영토가 보인다. "저걸 내 것으로 만들어야 해." |
| **② 물량의 카타르시스** | 중반 (3~10분) | 성장의 쾌감 + 시각적 장관 | 점령한 행성들에서 쏟아지는 수십~수백 기의 함선. 푸른 레이저와 붉은 레이저가 교차하는 전장. 내가 키운 함대가 우주를 가로지르며 적을 압도하는 장면. |
| **③ 전략적 역전** | 후반 (10~20분) | 긴장감 + 판단의 성취감 | 적 AI도 끊임없이 병력을 키워 역습해 온다. 무분별한 확장은 방어의 빈틈을 만든다. "지금 공격? 아직 수비?" — 고레벨 적 행성을 빼앗으며 보너스 골드를 챙기고 전세를 역전시키는 쾌감. |

### 이 게임이 '아닌' 것 (What This Game Is NOT)

| ❌ 이것이 아닙니다 | ✅ 이것입니다 |
|---|---|
| 빌드 오더를 외워야 하는 하드코어 RTS | 직관적으로 "선택 → 명령"만 하면 되는 라이트 RTS |
| 유닛 하나하나를 마이크로 컨트롤하는 게임 | 대규모 함대를 한 번에 지휘하며 전체 전선을 관리하는 게임 |
| 30분 이상 걸리는 장기전 | 10~20분 안에 한 판이 마무리되는 숏 세션 전략 |
| 고퀄리티 3D 그래픽의 AAA 게임 | 미니멀한 도형과 네온 컬러로 표현한 '망원경으로 본 은하 전쟁'의 미학 |

## 2. 아이디어 연상 과정

### 2.1. 연상법: 레퍼런스 변형 + 메커니즘 조합

이 게임의 아이디어는 **기존 레퍼런스의 장점을 추출하고, 서로 다른 장르의 메커니즘을 조합**하여 도출했습니다.

```

Galcon (갤콘)                   스타크래프트                     영토 전략 보드게임 (리스크)
  │                               │                               │
  │ "행성 점령의                   │ "부대 지정,                    │ "빈 영토를 하나씩
  │  직관적 쾌감"                  │  컨트롤 그룹의                 │  물들이는 정복 욕구"
  │                               │  전략적 손맛"                  │
  └───────────┬───────────────────┴───────────┬───────────────────┘
              │                               │
              ▼                               ▼
    ┌─────────────────┐             ┌─────────────────┐
    │ 추출: 심플한     │             │ 추출: RTS        │
    │ 행성 점령 루프   │             │ 컨트롤의 손맛    │
    └────────┬────────┘             └────────┬────────┘
             │                               │
             └───────────┬───────────────────┘
                         │
                         ▼
              ┌──────────────────────┐
              │  Space Battle Fleet  │
              │ 간결한 점령 루프     │
              │ + 정통 RTS 컨트롤    │
              │ + 영토 정복의 시각적 │
              │   만족감             │
              └──────────────────────┘
```

### 2.2. 각 레퍼런스에서 가져온 것과 버린 것

| 레퍼런스 | 가져온 것 (Adopt) | 버린 것 (Drop) | 이유 |
|----------|------------------|----------------|------|
| **Galcon** | 행성 간 병력 이동의 직관성, 숫자 물량전 | 클릭 한 번으로 끝나는 단순 조작 | 전략적 깊이 부족 → RTS 컨트롤로 보완 |
| **스타크래프트** | 부대 지정(Ctrl+숫자), 드래그 선택, 랠리 포인트, 미니맵 | 복잡한 빌드 오더, 자원 채집, 건물 건설 | 진입장벽 ↓ → 생산은 행성에서 자동 |
| **리스크 (보드게임)** | 영토를 하나씩 색칠해가는 정복감, 줄다리기 전선 | 턴제 플레이, 주사위 기반 전투 | 실시간 + 액션성 → 레이저 교전으로 전환 |

### 2.3. 독자적으로 추가한 메커니즘

| 메커니즘 | 내용 | 추가 이유 |
|----------|------|-----------|
| **행성 레벨 업그레이드** | 골드를 투자해 생산 속도를 강화 | "넓게 vs 깊게"의 전략 분기점 제공 |
| **점령 보너스 골드** | 적의 고레벨 행성을 빼앗으면 큰 보상 | 공격적 플레이에 대한 리스크/리턴 보상 체계 |
| **체인 랠리 포인트** | 행성을 경유지로 연결하는 보급선 설정 | 전선이 넓어질수록 "물류 관리"의 전략적 재미 부여 |
| **3종 함선 체계** | 소형(빠름)/중형(밸런스)/대형(강력) | 단일 유닛 스팸 방지, 상황별 편성 판단 유도 |

## 3. 컨셉 방향성 및 타겟팅

### 3.1. 컨셉 유형: 문제해결 중심 컨셉

이 게임은 **타겟 유저가 겪는 구체적인 문제를 해결**하기 위해 설계되었습니다.

### 3.2. 타겟 유저가 겪는 문제

**타겟: "RTS를 좋아하지만 진입장벽에 좌절하는 캐주얼~미드코어 게이머"**

| 문제 (Pain Point) | 구체적 상황 | 본 게임의 해결책 |
|-------------------|------------|-----------------|
| **① RTS의 높은 진입장벽** | 스타크래프트, 에이지 오브 엠파이어 등 정통 RTS는 빌드 오더, 자원 관리, 건물 배치, 테크트리 등 학습해야 할 요소가 너무 많다. 유튜브에서 대규모 전투 영상을 보면 재밌어 보이지만, 막상 플레이하면 멀티태스킹에 압도당한다. | 자원 채집·건물 건설·테크트리를 완전히 제거. 행성이 자동으로 유닛을 생산하므로, 플레이어는 **"어디를 공격할까"**와 **"무엇을 업그레이드할까"** 두 가지 판단에만 집중하면 된다. |
| **② 긴 플레이 타임** | 한 판에 30분~1시간이 걸리는 기존 RTS는 바쁜 현대인에게 부담. 점심시간이나 잠깐의 여유에 전략 게임을 즐기기 어렵다. | 10~20분 내 한 판 완료. 맵이 매번 랜덤으로 생성되어 짧은 세션으로도 매 판 다른 전략적 경험을 제공한다. |
| **③ 물량전의 쾌감을 느끼기 어려움** | 기존 RTS에서 대규모 물량전까지 도달하려면 10분 이상의 자원 축적 시간이 필요. 초보자는 그 전에 이미 패배하는 경우가 많다. | 게임 시작 2~3분이면 수십 기의 함선이 생산된다. 초반부터 물량전의 시각적 쾌감을 경험할 수 있고, 행성을 점령할수록 생산 라인이 기하급수적으로 증가한다. |
| **④ 설치 및 환경 장벽** | 게임 클라이언트를 다운받고 설치하는 과정이 번거롭다. 회사 PC나 공용 컴퓨터에서는 설치 자체가 불가능한 경우도 많다. | 웹 브라우저만 있으면 즉시 플레이 가능. 설치 불필요. URL 하나로 누구에게나 공유 가능. |

## 6. 자원 시스템

### 6.1. 인구수

- **최대치:** 500 (플레이어·AI 동일)
- 함선을 생산할 때마다 유닛별 인구(1~3)를 소비하고, 유닛 파괴 시 반환
- 인구 최대치 도달 시 전 행성 생산 일시 중지
- **설계 의도:** 무한 물량 방지. "지금 소형을 대량으로 뽑을까, 대형 소수를 뽑을까?"라는 편성 고민을 유발

### 6.2. 골드

| 획득 경로 | 보상 |
|-----------|------|
| 적 요격기(소형) 격파 | +1G |
| 적 구축함(중형) 격파 | +2G |
| 적 전함(대형) 격파 | +3G |
| **행성 점령** | **점령 전 행성 레벨 × 20G** |

- **사용처:** 아군 행성 레벨 업그레이드
- **설계 의도:** 전투에 적극 참여할수록, 고레벨 행성을 공략할수록 경제적 이점. 수비만 하는 플레이를 지양하고 공격적 확장에 보상을 부여.

### 6.3. 승패 조건

- **승리:** 적 소유 행성 0개
- **패배:** 플레이어 소유 행성 0개

## 7. 행성 시스템

행성은 **기지**(유닛 생산), **목표**(점령 대상), **투자처**(업그레이드 대상)의 세 가지 역할을 동시에 수행합니다.

### 7.1. 행성 종류

| 구분 | 색상 | HP | 유닛 생산 | 역할 |
|------|------|----|-----------|------|
| **아군 행성** | 파란색 | 500 | O | 플레이어의 함선 생산 기지 |
| **적군 행성** | 빨간색 | 500 | O | AI의 함선 생산 기지 |
| **중립 행성** | 회색 | 200 | X | 점령하면 아군 기지로 전환 |

### 7.2. 점령 시스템

1. 함선이 적/중립 행성의 HP를 0으로 깎음
2. **마지막 타격을 가한 진영**의 소유로 전환
3. 점령 시: HP 전량 회복, 레벨 1 초기화, 기본 유닛(요격기) 생산 시작
4. **점령 보너스:** 기존 행성 레벨 × 20G 지급

### 7.3. 행성 레벨 업그레이드

| 레벨 | 업그레이드 비용 | 생산 시간 단축 효과 |
|------|----------------|-------------------|
| 1 → 2 | 50G | 10% 단축 |
| 2 → 3 | 100G | 20% 단축 |
| 3 → 4 | 150G | 30% 단축 |
| ... | 레벨 × 50G | 최대 90% 단축 |

- **설계 의도:** "행성을 많이 점령할까 vs 핵심 거점을 집중 업그레이드할까"의 전략 분기. 넓게 확장해서 물량으로 밀어붙이는 전략과, 소수 행성을 극한까지 강화하는 엘리트 전략 모두 유효.

### 7.4. 랠리 포인트 (집결지)

- 아군 행성 선택 후 우클릭으로 목표 지정
- 대기 유닛 즉시 출격 + 이후 신규 생산 유닛도 자동 출격
- **체인 랠리:** 경유 행성에도 랠리 포인트가 있으면 멈추지 않고 연쇄 이동
- **설계 의도:** "생산 라인을 설정해두면 자동으로 전선에 병력이 공급되는" 전쟁 물류 관리 경험

### 7.5. 생산 게이지

- 행성 하단에 프로그레스 바로 현재 유닛 생산 진행 상황 실시간 표시
- 레벨에 따른 단축 효과 반영

## 8. 함선 (유닛) 시스템

3종의 함선이 각각 **속도·화력·내구**에서 뚜렷한 차별점을 가집니다.

### 8.1. 함선 스펙

| | 요격기 (소형) | 구축함 (중형) | 전함 (대형) |
|---|---|---|---|
| **외형** | ▲ 삼각형 | ■ 사각형 | ● 원형 |
| **체력 / 공격력** | 30 / 5 | 80 / 15 | 200 / 40 |
| **이동속도** | 120 (빠름) | 80 (보통) | 40 (느림) |
| **생산시간** | 2.4초 | 4.8초 | 7.2초 |
| **인구** | 1 | 2 | 3 |
| **처치 보상** | 1G | 2G | 3G |
| **전략적 역할** | 빠른 기동, 중립 선점, 빈집 털이 | 공방 밸런스의 주력함 | 느리지만 강력, 방어선 돌파 |

### 8.2. 전투 시스템

- **자동 교전:** 사거리 내 적 감지 시 이동 정지 → 0.5초 간격 레이저 공격
- **타겟 우선순위:** 적 유닛 > 적/중립 행성
- **시각 피드백:** 진영 색상 레이저 이펙트 발사, 피해 유닛 체력바 표시

### 8.3. 편대 행동

- **겹침 방지:** 유닛 간 일정 거리 이하 시 분리력 작용 → 자연스러운 원형 진형 형성
- **방향 표현:** 요격기(삼각형)는 이동 방향으로 회전하여 편대 진행 방향 시각화

## 12. AI 시스템

### 설계 철학

> **"끊임없이 압박하되, 완벽하지 않은 상대"**

최적의 판단을 하는 AI가 아니라, 무작위성과 규칙을 섞어 **"대응 가능하지만 방심하면 위험한"** 긴장감을 유지합니다.

### 행동 패턴

| 행동 | 주기/조건 | 내용 |
|------|-----------|------|
| **전장 분석** | 5초마다 | 전체 상황 판단 (펄스 단위) |
| **생산 변경** | 펄스당 30% | 요격기 50% / 구축함 30% / 전함 20% 확률 변경 |
| **공격 파견** | 대기 유닛 5기 초과 시 | 잉여 유닛을 무작위 적대 행성으로 파견 |
| **행성 업그레이드** | 펄스당 15% | 확률 기반 자동 레벨업 (최대 Lv.10) |

- **설계 의도:** 무작위 생산 변경으로 단일 유닛 대응 불가. 지속적 공격 파견으로 방어 소홀 불가.
