# 스페이스 배틀 플릿 (Space Battle Fleet) 최종 기획서

---

## 1. 게임 개요

- **게임명:** 스페이스 배틀 플릿 (Space Battle Fleet)
- **장르:** 2D 탑뷰 실시간 우주 영토 점령 전략 시뮬레이션 (RTS)
- **목표:** 자신에게 주어진 시작 행성을 기반으로 자원과 병력을 모아, 맵 상의 모든 적(AI) 행성을 점령하고 우주를 통일하는 것.

### 핵심 재미 요소
- 복잡한 빌드 오더 없이 직관적인 생산과 '어택땅' 위주의 대규모 물량전
- 행성 업그레이드와 3가지 병종을 활용한 전략적 가위바위보 상성
- 부대 지정, 다중 선택, 랠리 포인트 등 정통 RTS의 컨트롤 손맛

---

## 2. 핵심 자원 및 시스템

### 2.1. 인구수 (Population)
- **최대 인구수:** 500 (플레이어 및 AI 동일하게 적용)
- **시스템:** 병력을 생산할 때마다 각 유닛에 배정된 인구수(1~3)를 차지하며, 유닛이 파괴되면 인구수가 반환됩니다. 인구수가 최대치에 도달하면 행성에서의 생산이 일시 중지됩니다.

### 2.2. 자금 (Gold)
- **획득처:** 적(AI)의 유닛을 파괴할 때마다 유닛 체급에 따라 골드를 획득합니다. (소형 1G / 중형 2G / 대형 3G)
- **행성 점령 보너스:** 행성 점령 시 점령 전 행성의 레벨 × 20G를 추가 획득합니다.
- **사용처:** 아군 소유 행성의 레벨 업그레이드에 사용됩니다.

### 2.3. 승패 조건 (Phase System)
- **승리 (VICTORY):** 적(AI) 소유의 행성이 맵 상에 0개가 되면 즉시 승리합니다.
- **패배 (DEFEAT):** 플레이어 소유의 행성이 맵 상에 0개가 되면 즉시 패배합니다.

## 3. 맵 구성 및 초기 배치

### 3.1. 맵 사양
- **맵 크기:** 3000 × 2000 (px)
- **배경:** 600개의 별(Stars)이 패럴랙스(시차) 효과로 배경을 장식
- **그리드:** 200px 간격의 그리드 라인 (rgba(255,255,255,0.03))

### 3.2. 초기 행성 배치
| 진영 | 위치 | HP |
|------|------|----|
| 플레이어 행성 1 | (350, 650) | 500 |
| 플레이어 행성 2 | (350, 1350) | 500 |
| AI 행성 1 | (2650, 650) | 500 |
| AI 행성 2 | (2650, 1350) | 500 |
| 중립 행성 ×15 | 랜덤 (최소 180px 간격) | 200 |

- 중립 행성은 맵 가장자리 200px 여백을 두고 배치됩니다.
- 행성 간 최소 거리 180px을 보장하기 위해 while 루프로 유기적 배치합니다 (최대 5000회 시도).

## 4. 행성 (Planet) 및 영토 시스템

### 4.1. 행성 상태 및 소유권
- **아군 행성 (파란색):** 플레이어의 유닛이 생산되는 기지입니다.
- **적군 행성 (붉은색):** AI의 유닛이 생산되는 기지입니다.
- **중립 행성 (회색):** 기본 내구도(200 HP)를 가지며, 점령 전까지는 유닛을 생산하지 않습니다.

### 4.2. 점령 시스템
- 함선이 목표 행성의 HP를 0으로 깎으면 즉시 **마지막 공격을 가한 진영의 소유**로 넘어갑니다.
- 점령 시: HP가 최대치(maxHp)로 회복, 레벨 1로 초기화, 기본 유닛(요격기) 생산 시작
- 점령 시 점령 전 행성의 레벨 × 20G가 점령자(플레이어)에게 보너스로 지급됩니다.
- 해당 행성에 주둔 중이던 적 유닛의 `homePlanetId`가 null로 초기화됩니다.

### 4.3. 행성 레벨 및 업그레이드
- **기본 상태:** 모든 행성은 점령 시 1레벨로 시작합니다.
- **업그레이드 비용:** 현재 레벨 × 50G (예: 1레벨 → 2레벨은 50G, 2레벨 → 3레벨은 100G)
- **레벨업 효과:** 행성 레벨이 1 오를 때마다 해당 행성의 유닛 생산 시간이 10%씩 단축됩니다. (단축 최대치는 90%로 제한되어 0초 생산 방지)

### 4.4. 랠리 포인트 (집결지)
- 아군 행성을 선택하고 빈 우주 공간이나 타겟 행성을 우클릭하면 랠리 포인트가 지정됩니다.
- **작동 방식:**
  - 지정 시 해당 행성에 대기 중이던 모든 유닛이 즉시 목표로 출격합니다.
  - 이후 해당 행성에서 새로 스폰되는 유닛들도 멈추지 않고 즉시 목표 지점으로 날아갑니다.
- **체인 랠리:** 유닛이 아군 행성에 도착했을 때, 해당 행성에도 랠리 포인트가 설정되어 있으면 멈추지 않고 다음 랠리 포인트로 이동합니다.
- **시각적 표시:** 선택된 행성의 랠리 포인트는 노란색 점선과 마커(●)로 표시됩니다.

### 4.5. 행성 렌더링
- 반지름: 30px
- RadialGradient로 입체감 있는 구체 렌더링
- 그림자(glow) 효과 (shadowBlur: 25)
- 상단 HP 바 (50×5px) 표시
- 중앙에 "Lv{레벨}" 텍스트
- 하단에 현재 생산 유닛명 표시
- 하단에 생산 진행 게이지 바 (36×4px) 표시
- 선택 시 흰색 링(radius + 6px) 표시

## 5. 유닛 (Unit) 기획

유닛은 크게 3종류로 나뉘며, 각기 다른 외형(도형), 능력치, 생산 속도, 요구 인구수를 가집니다.

### 5.1. 유닛 스탯 표

| 속성 | 요격기 (타입 1) | 구축함 (타입 2) | 전함 (타입 3) |
|------|----------------|----------------|--------------|
| **외형** | ▲ 삼각형 | ■ 사각형 | ● 원형 |
| **HP** | 30 | 80 | 200 |
| **공격력** | 5 | 15 | 40 |
| **이동속도** | 120 px/s | 80 px/s | 40 px/s |
| **생산시간** | 2.4초 | 4.8초 | 7.2초 |
| **충돌반경** | 6px | 9px | 14px |
| **인구** | 1 | 2 | 3 |
| **처치 골드** | 1G | 2G | 3G |

### 5.2. 부대 AI 및 진형 유지 알고리즘
- **자동 교전:** 사거리(120px) 내에 적 유닛이나 적대 행성이 포착되면 이동을 멈추고 0.5초의 쿨타임마다 레이저 공격(파티클 이펙트)을 가합니다. **적 유닛을 우선적으로** 타겟팅합니다.
- **겹침 방지 (Separation):** 수백 기의 유닛이 한 점으로 뭉치는 것을 방지하기 위해 유닛 간 거리를 계산하여 서로 밀어내는 알고리즘이 적용되어 있습니다.
  - 분리 거리: 35px 이내 유닛에 대해 계산
  - 분리 강도: 100
  - `Math.abs(dx) > 35` 인 경우 `Math.hypot` 연산 생략 (성능 최적화)
- **요격기 방향 표시:** 요격기는 이동 방향을 향해 삼각형이 회전합니다 (`Math.atan2(vy, vx)`).
- **유닛 체력바:** 피해를 입은 유닛만 머리 위에 HP 바가 표시됩니다.

### 5.3. 유닛 스폰
- 행성 주위 랜덤 각도, 행성 반지름 + 유닛 반지름 + 8px 거리에 스폰
- 스폰 즉시 랠리 포인트가 설정되어 있으면 바로 출발

## 6. 조작 및 UI 시스템

### 6.1. 마우스 컨트롤
- **좌클릭 (단일):** 아군 행성 단일 선택 (유닛 선택 해제)
- **좌클릭 드래그:** 녹색 선택 박스로 영역 내 다수의 아군 유닛 선택 (행성 선택 해제)
- **더블클릭:** 아군 행성 클릭 시 300ms 이내 동일 행성 재클릭하면 화면 내 보이는 모든 아군 행성 동시 선택
- **우클릭 (행성 선택 시):** 랠리 포인트 지정 + 대기 유닛 즉시 출격
- **우클릭 (유닛 선택 시):** 이동/공격 명령 + 녹색 이동 마커 파티클 표시
- **마우스 휠:** 줌 인/아웃 (0.4배 ~ 1.5배, 단계: 0.1). 화면 중심 기준 확대/축소

### 6.2. 키보드 컨트롤
- **WASD / 방향키:** 전장 카메라 이동 (속도: 1000 px/s)
- **Shift + 1~4 또는 Ctrl + 1~4:** 현재 선택된 유닛들을 부대 슬롯에 저장 (스타크래프트 방식)
- **1~4:** 저장된 부대 호출 (사망한 유닛은 자동 필터링)

### 6.3. 엣지 스크롤링
- 마우스 커서가 화면 가장자리 30px 이내에 위치하면 해당 방향으로 자동 카메라 이동 (속도: 1000 px/s)

### 6.4. 미니맵 (Minimap) 시스템
- 우측 하단에 300×300 사이즈의 전체 맵 조망 미니맵
- 여백: 화면 가장자리에서 10px
- **행성:** 반지름 4px 원으로 진영 색상 표시
- **유닛:** 2×2px 점으로 진영 색상 표시
- **카메라 뷰포트:** 현재 보이는 영역을 흰색 사각형으로 표시
- **좌클릭 및 드래그:** 카메라가 클릭/드래그한 위치로 즉시 이동
- **우클릭:** 화면 이동 없이 월드 좌표로 변환하여 병력 이동 / 랠리 포인트 지정

### 6.5. HUD (헤드업 디스플레이)

#### 상단 좌측 - 자원 표시
- 골드 (⬡ {금액} G) 및 인구수 (♦ {현재} / 500) 표시
- 반투명 검정 배경 + 블러 효과

#### 상단 우측 - 유닛 선택 표시
- 유닛이 선택된 경우 "유닛 선택: {N}기" 표시

#### 하단 좌측 - 부대 지정 슬롯 (1~4)
- 4개의 버튼 (14×14px)
- 유닛이 저장된 슬롯: 파란색 배경 + 유닛 수 표시
- 빈 슬롯: 반투명 + "비어있음" 텍스트
- 클릭으로도 부대 호출 가능

#### 하단 중앙 - 컨트롤 패널 (행성 선택 시)
- 선택된 행성 개수 표시
- 단일 선택 시: HP/MaxHP, 레벨 상세 정보 표시
- **생산 변경 버튼:** ▲ 요격기 / ■ 구축함 / ● 전함 (현재 생산 중인 유닛 강조)
- **레벨 업그레이드 버튼:** 비용 표시 + 골드 부족 시 비활성화

## 7. 이펙트 및 파티클 시스템

### 7.1. 레이저 이펙트
- 유닛 공격 시 공격자 → 피격자를 잇는 직선 레이저
- 진영 색상의 밝은(light) 컬러 사용
- 수명: 0.15초, 페이드아웃(alpha = life / maxLife)

### 7.2. 이동 마커
- 유닛에게 이동 명령 시 목표 지점에 표시
- 녹색 원형 링 + X 마크
- 수명: 0.8초, 확장 + 페이드아웃 애니메이션

### 7.3. 배경 별(Stars)
- 600개의 별이 맵 전역에 분포
- 크기: 0.5 ~ 2.5px
- 밝기: 0.2 ~ 0.8 (globalAlpha)
- 색상: 흰색(80%), 노란빛(10%), 푸른빛(10%)
- 패럴랙스 계수: 별 × 0.7, 카메라 × 0.3 (카메라 이동 시 느린 속도로 움직임)

## 8. 인공지능 (AI) 시스템

### 8.1. 행동 주기 (Pulse)
- AI는 **5초에 한 번씩** 전장 상황을 판단하고 명령을 내립니다.

### 8.2. 생산 무작위화
- 행동 주기마다 **30%의 확률**로 AI 행성의 생산 유닛이 변경됩니다.
- 확률 분포: 요격기 50%, 구축함 30%, 전함 20%
- 이를 통해 AI도 가위바위보 조합을 섞어서 공격해 옵니다.

### 8.3. 공격 판정
- 특정 AI 행성에 대기 중인 잉여 유닛이 **5기를 초과**하면, 즉시 해당 유닛들을 맵 상의 무작위 적대 행성(플레이어 또는 중립)으로 파견합니다.

### 8.4. AI 자동 업그레이드
- 행동 주기마다 각 AI 행성에 대해 **15%의 확률**로 레벨이 1 상승합니다.
- 최대 레벨 10까지 자동 업그레이드됩니다.
- (AI는 골드 시스템을 사용하지 않고 확률 기반으로 업그레이드)

## 11. 기술 스택 및 아키텍처

### 11.1. 기술 스택
- **프레임워크:** React 18 (Functional Component) + HTML5 `<canvas>`
- **스타일링:** Tailwind CSS (UI 레이어 및 오버레이용)
- **빌드:** CDN 기반 (react, react-dom, babel-standalone, tailwindcss)
- **에셋:** 외부 이미지 없음 — Canvas 2D API로 모든 도형, 선, 색상, 그림자 직접 렌더링
- **파일 구조:** index.html 단일 파일

### 11.2. 상태 관리 아키텍처

#### 게임 코어 상태 (useRef - 렌더링 독립)
초당 60프레임 이상 업데이트되는 모든 물리 연산과 게임 데이터는 `useRef`에 담아 React 리렌더링을 방지합니다.
- `planets`, `units`, `particles`, `stars`: 게임 엔티티 배열
- `camera`: `{ x, y, w, h, zoom }` (현재 화면 좌표 및 줌)
- `mouse`, `keys`: 입력 상태 및 드래그 좌표 추적
- `controlGroups`: 부대 지정 정보 (`{ 1: Set, 2: Set, 3: Set, 4: Set }`)
- `playerGold`, `playerPop`, `aiPop`: 자원 및 인구 카운터
- `aiTimer`, `uiSyncTimer`: 타이머 변수

- `phase`, `gameResult`
- `playerGold`, `playerPop`
- `selectedPlanets`, `selectedUnitsCount`

### 11.4. 게임 루프 구조
```
requestAnimationFrame(gameLoop)
  └─ update(dt)
       ├─ updateCamera(dt)      // WASD, 엣지스크롤, 클램핑
       ├─ updatePlanets(dt)     // 유닛 생산 (레벨 보정 적용)
       ├─ updateUnits(dt)       // 분리, 전투, 이동, 사망 처리
       ├─ updateParticles(dt)   // 레이저·마커 수명 관리
       ├─ updateAI(dt)          // 5초 펄스 시스템
       ├─ checkWinLose()        // 승패 판정
       └─ syncUI(dt)            // 0.1s 주기 UI 동기화
  └─ render(ctx)
       ├─ 배경 (별 + 그리드)
       ├─ 랠리 포인트 라인
       ├─ 파티클 (레이저, 이동 마커)
       ├─ 행성 (글로우 + 그래디언트 + HP바 + 텍스트)
       ├─ 유닛 (도형 + HP바)
       ├─ 선택 박스
       └─ 미니맵
```
