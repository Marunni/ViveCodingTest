# I. 게임 소개

## 2. 핵심 유저 경험 정의

### 한 문장 정의

> **"복잡한 조작 없이 수백 기의 함대를 지휘하며, '내 영토가 넓어지는 정복의 쾌감'과 '한 번의 판단 착오로 전세가 뒤집히는 줄다리기 긴장감'을 동시에 느끼는 경험."**

이 한 줄이 게임의 모든 설계 결정의 출발점입니다. 이하 모든 시스템, 캐릭터, 그래픽은 이 경험을 전달하기 위해 존재합니다.

### 감정 여정 (Emotional Journey)

| 단계 | 게임 시점 | 제공하는 감정 | 구체적 상황 |
|------|-----------|--------------|-------------|
| **① 고독한 개척** | 초반 (0~3분) | 미지에 대한 긴장 + 탐색 욕구 | 광활한 우주의 왼쪽 끝, 단 2개의 행성만 가진 채 시작. 카메라를 움직이면 회색 미지의 행성들과 저 멀리 붉게 빛나는 적의 영토가 보인다. "저걸 내 것으로 만들어야 해." |
| **② 물량의 카타르시스** | 중반 (3~10분) | 성장의 쾌감 + 시각적 장관 | 점령한 행성들에서 쏟아지는 수십~수백 기의 함선. 푸른 레이저와 붉은 레이저가 교차하는 전장. 내가 키운 함대가 우주를 가로지르며 적을 압도하는 장면. |
| **③ 전략적 역전** | 후반 (10~20분) | 긴장감 + 판단의 성취감 | 적 AI도 끊임없이 병력을 키워 역습해 온다. 무분별한 확장은 방어의 빈틈을 만든다. 고레벨 적 행성을 빼앗으며 보너스 골드를 챙기고 전세를 역전시키는 쾌감. |

### 핵심 재미 요소 3가지

1. **직관적 물량전** — 복잡한 빌드 오더 없이 "선택 → 명령"만으로 대규모 함대 전투
2. **전략적 상성** — 3가지 병종(요격기/구축함/전함)의 속도·화력·내구 가위바위보
3. **정통 RTS 손맛** — 부대 지정, 드래그 선택, 랠리 포인트, 미니맵 등 컨트롤의 쾌감

## 3. 타겟 유저

### 유저 페르소나

| 항목 | 내용 |
|------|------|
| **이름** | 김전략 (가상) |
| **나이** | 25~35세 |
| **직업** | 직장인 / 대학생 |
| **게임 성향** | 미드코어. 모바일 전략 게임(클래시 로얄, 부족전쟁)을 즐기고, 유튜브로 스타크래프트 프로리그를 시청. 직접 플레이하면 빌드 오더를 못 외워서 금방 포기. |
| **플레이 환경** | 점심 시간, 퇴근 후, 주말 30분 여유 시간 |
| **핵심 니즈** | "생각은 전략적으로, 조작은 간단하게. 짧은 시간에 대규모 전투의 짜릿함을 느끼고 싶다." |

### 타겟 유저의 Pain Point → 해결 전략

| 문제 (Pain Point) | 구체적 상황 | 본 게임의 해결책 |
|-------------------|------------|-----------------|
| **① RTS의 높은 진입장벽** | 스타크래프트 등은 빌드 오더, 자원 관리, 건물 배치, 테크트리 학습 요소가 과다 | 자원 채집·건물 건설·테크트리 완전 제거. **"어디를 공격할까"**와 **"무엇을 업그레이드할까"** 두 가지 판단에만 집중 |
| **② 긴 플레이 타임** | 한 판 30분~1시간의 기존 RTS는 바쁜 현대인에게 부담 | **10~20분** 내 한 판 완료. 매판 랜덤 맵으로 짧은 세션에서도 매번 다른 전략 경험 |
| **③ 물량전까지 너무 오래 걸림** | 기존 RTS에서 대규모 전투까지 10분+ 자원 축적 필요 | 게임 시작 **2~3분이면** 수십 기의 함선 생산. 초반부터 물량전의 시각적 쾌감 |
| **④ 설치 및 환경 장벽** | 게임 클라이언트 다운로드·설치 과정이 번거로움 | **웹 브라우저만 있으면 즉시 플레이**. URL 하나로 공유 가능 |

## 2. 레퍼런스 분석: 가져온 것과 버린 것

| 레퍼런스 | 가져온 것 (Adopt) | 버린 것 (Drop) | 이유 |
|----------|------------------|----------------|------|
| **Galcon** | 행성 간 병력 이동의 직관성, 숫자 물량전 | 클릭 한 번으로 끝나는 단순 조작 | 전략적 깊이 부족 → RTS 컨트롤로 보완 |
| **스타크래프트** | 부대 지정(Ctrl+숫자), 드래그 선택, 랠리 포인트, 미니맵 | 복잡한 빌드 오더, 자원 채집, 건물 건설 | 진입장벽 ↓ → 생산은 행성에서 자동 |
| **리스크 (보드게임)** | 영토를 하나씩 색칠해가는 정복감, 줄다리기 전선 | 턴제 플레이, 주사위 기반 전투 | 실시간 + 액션성 → 레이저 교전으로 전환 |

## 3. 독자적 메커니즘

기존 레퍼런스에는 없는, 본 게임만의 고유 시스템을 추가하여 전략적 깊이를 확보했습니다.

| 메커니즘 | 내용 | 추가 이유 (컨셉 연결) |
|----------|------|----------------------|
| **행성 레벨 업그레이드** | 골드를 투자해 생산 속도 강화 (최대 90% 단축) | "넓게 점령 vs 깊게 강화"의 전략 분기점 → **판단의 긴장감** |
| **점령 보너스 골드** | 적의 고레벨 행성을 빼앗으면 레벨×20G 보상 | 공격적 플레이의 리스크/리턴 → **역전의 카타르시스** |
| **체인 랠리 포인트** | 행성을 경유지로 연결하는 보급선 설정 | 전선이 넓어질수록 "물류 관리" → **정복의 확장감** |
| **3종 함선 체계** | 소형(빠름)/중형(밸런스)/대형(강력) 상성 | 단일 유닛 스팸 방지, 편성 판단 유도 → **전략적 재미** |

## 4. 이 게임이 '아닌' 것 vs '인' 것

| ❌ 이것이 아닙니다 | ✅ 이것입니다 |
|---|---|
| 빌드 오더를 외워야 하는 하드코어 RTS | 직관적으로 "선택 → 명령"만 하면 되는 라이트 RTS |
| 유닛 하나하나를 마이크로 컨트롤하는 게임 | 대규모 함대를 한 번에 지휘하며 전체 전선을 관리하는 게임 |
| 30분 이상 걸리는 장기전 | 10~20분 안에 한 판이 마무리되는 숏 세션 전략 |
| 고퀄리티 3D 그래픽의 AAA 게임 | 미니멀한 도형과 네온 컬러로 표현한 '망원경으로 본 은하 전쟁'의 미학 |

## 5. 컨셉 → 설계로의 의도 연결

```
핵심 유저 경험
"정복의 쾌감 + 줄다리기 긴장감"
         │
         ├──▶ [행성 점령 시스템] ─── 영토를 색칠하는 정복의 쾌감
         │
         ├──▶ [자동 생산 + 랠리] ─── 복잡함 제거 → 전략 판단에 집중
```

# III. 상세 기획 (게임 설계)

> 이 섹션은 바이브코딩으로 즉시 구현 가능할 수준의 상세 명세를 포함합니다.

## 1. 핵심 게임 루프

```
    ┌──────────────────────────────────────────────┐
    │   ① 행성에서 함선이 자동으로 생산된다            │
    └──────────┬───────────────────────────────────┘
               ▼
    ┌──────────────────────────────────────────────┐
    │   ② 플레이어가 병력을 모아 목표 행성으로 출격     │
    └──────────┬───────────────────────────────────┘
               ▼
    ┌──────────────────────────────────────────────┐
    │   ③ 전투 발생 → 적 유닛 격파 시 골드 획득        │
    └──────────┬───────────────────────────────────┘
               ▼
    ┌──────────────────────────────────────────────┐
    │   ④ 행성 점령 → 점령 보너스 골드 획득            │
    └──────────┬───────────────────────────────────┘
               ▼
    ┌──────────────────────────────────────────────┐
    │   ⑤ 골드로 행성 업그레이드 → 생산 효율 강화      │
    └──────────┬───────────────────────────────────┘
               │              ↑ 반복
               └──────────────┘
```

**설계 의도:** 플레이어의 의사결정을 **"어디를 공격할까"**와 **"무엇을 업그레이드할까"** 두 가지로 압축. 복잡한 빌드 오더 없이 전략적 판단만으로 게임이 진행됩니다.

## 2. 자원 시스템

### 2.1. 인구수 (Population)

| 항목 | 값 |
|------|----|
| **기본 인구 상한** | 100 |
| **레벨당 추가 인구** | +5 (소유 행성들의 레벨 합산) |
| **인구 소비** | 유닛 생산 시 유닛별 인구(1~3) 차지 |
| **인구 반환** | 유닛 파괴 시 즉시 반환 |
| **상한 도달 시** | 전 행성 생산 일시 중지 |

**설계 의도:** 무한 물량 방지. "소형 대량 vs 대형 소수" 편성 고민 유발.

### 2.2. 골드 (Gold)

| 획득 경로 | 보상 |
|-----------|------|
| 적 요격기(소형) 격파 | +1G |
| 적 구축함(중형) 격파 | +2G |
| 적 전함(대형) 격파 | +3G |
| **행성 점령** | **점령 전 행성 레벨 × 20G** |

- **사용처:** 아군 행성 레벨 업그레이드 (비용: 현재 레벨 × 50G)
- **설계 의도:** 전투에 적극 참여할수록, 고레벨 행성을 공략할수록 경제적 이점. 수비만 하는 플레이를 지양하고 공격적 확장에 보상을 부여.

### 2.3. 승패 조건

| 조건 | 트리거 |
|------|--------|
| **승리 (VICTORY)** | 적(AI) 소유 행성이 0개가 되면 즉시 승리 |
| **패배 (DEFEAT)** | 플레이어 소유 행성이 0개가 되면 즉시 패배 |

## 3. 행성 (Planet) 시스템

### 3.1. 행성 종류

| 구분 | 색상 | HP | 유닛 생산 | 역할 |
|------|------|----|-----------|------|
| **아군 행성** | 파란색 (#4499ff) | 1000 | O | 플레이어의 함선 생산 기지 |
| **적군 행성** | 빨간색 (#ff4455) | 1000 | O | AI의 함선 생산 기지 |
| **중립 행성** | 회색 (#888888) | 400 | X | 점령하면 아군 기지로 전환 |

### 3.2. 점령 시스템

1. 함선이 적/중립 행성의 HP를 0으로 깎음
2. **마지막 타격을 가한 진영**의 소유로 전환
3. 점령 시: HP 전량 회복, 레벨 1 초기화, 기본 유닛(요격기) 생산 시작
4. **점령 보너스:** 기존 행성 레벨 × 20G 지급
5. 해당 행성에 주둔 중이던 적 유닛의 `homePlanetId`가 null로 초기화

### 3.3. 행성 레벨 업그레이드

| 레벨 업 | 비용 | 누적 생산 단축 |
|---------|------|---------------|
| 1 → 2 | 50G | 10% |
| 2 → 3 | 100G | 20% |
| 3 → 4 | 150G | 30% |
| N → N+1 | N × 50G | N × 10% (최대 90%) |

**설계 의도:** "행성을 많이 점령할까 vs 핵심 거점을 집중 업그레이드할까"의 전략 분기.

### 3.4. 랠리 포인트 (집결지)

- 아군 행성 선택 후 우클릭으로 목표 지정
- **즉시 출격:** 대기 유닛이 즉시 목표로 출발
- **자동 출격:** 이후 신규 생산 유닛도 스폰 즉시 출발
- **체인 랠리:** 경유 행성에도 랠리 포인트가 있으면 멈추지 않고 연쇄 이동
- **시각적 표시:** 노란색 점선 + 목표 마커(●)

**설계 의도:** "생산 라인을 설정해두면 자동으로 전선에 병력이 공급되는" 전쟁 물류 관리 경험.

# V. 기술 구현 요약

## 1. 기술 스택

| 기술 | 역할 |
|------|------|
| **React 18** | UI 컴포넌트 (상태 관리, HUD 렌더링) |
| **HTML5 Canvas** | 게임 월드 렌더링 (행성, 유닛, 이펙트) |
| **Tailwind CSS** | UI 오버레이 스타일링 |
| **CDN 기반 로드** | react, react-dom, babel-standalone, tailwindcss |

## 2. 아키텍처 핵심

- **단일 파일 (index.html):** 외부 에셋 없이 모든 그래픽을 Canvas 2D API로 직접 렌더링
- **게임 코어 (useRef):** 초당 60+ 프레임 처리되는 물리 연산은 React 리렌더링에서 독립
- **UI 동기화 (useState):** 0.1초마다 게임 상태를 UI에 반영하여 성능 확보
- **requestAnimationFrame 게임 루프:** update → render 순서로 매 프레임 실행

## 3. 바이브코딩 접근성

본 게임은 **AI에게 프롬프트를 입력하여 즉시 코딩을 진행할 수 있는 수준**으로 설계되었습니다.

- 모든 변수와 상수가 명확하게 정의됨 (상수 테이블 참조)
- 엔티티 자료 구조(Planet, Unit)가 JSON 형태로 명세됨
- 게임 루프 구조가 함수 단위로 분리되어 프롬프트 가능
- 단일 HTML 파일이므로 별도 빌드 환경 불필요

### 엔티티 자료 구조 (바이브코딩 참조용)

**Planet 객체:**
```js
{
  id: string,           // 고유 ID
  x: number,            // 월드 X 좌표
  y: number,            // 월드 Y 좌표
  radius: 30,           // 행성 반지름 (고정)
  owner: 0 | 1 | 2,     // 0=중립, 1=플레이어, 2=AI
  hp: number,           // 현재 HP
  maxHp: number,        // 최대 HP (중립 400, 진영 1000)
  level: number,        // 1부터 시작
  productionType: 1|2|3, // 생산 유닛 타입
  spawnTimer: number,   // dt 누적
  rallyPoint: {x, y} | null,
  rallyPointPlanetId: string | null
}
```

**Unit 객체:**
```js
{
  id: string,           // 고유 ID
  owner: 1 | 2,         // 소유 진영
  type: 1 | 2 | 3,      // 유닛 타입
  x: number, y: number, // 월드 좌표
  vx: number, vy: number, // 속도 벡터
  hp: number, maxHp: number,
  state: 'IDLE' | 'MOVING' | 'IDLE_FIELD',
  homePlanetId: string | null,
  targetPlanetId: string | null,
  targetX: number | null,
  targetY: number | null,
  attackCooldown: number,  // 0.5초 간격
  attackTargetId: string | null
}
```
